Final Architecture

The final model is a decoder-only causal transformer containing four transformer blocks, four self-attention heads, a 160-dimensional embedding and a 128-token context window. It uses learned positional embeddings, pre-layer normalization, causal scaled dot-product attention and GELU feed-forward networks. The final checkpoint contains 1,360,320 trainable parameters, safely below the 2,000,000-parameter limit.

Lossless Multilingual Tokenizer

The original tokenizer represented every UTF-8 byte independently. The final tokenizer retains all 256 raw-byte tokens as universal fallback tokens and adds 64 frequent non-ASCII character tokens learned exclusively from train_corpus.txt. Therefore, arbitrary UTF-8 remains exactly representable, including unseen characters.

The tokenizer reduced the supplied corpus from 7,318,592 tokens to 5,716,436 tokens, a reduction of 21.80%. This provides more linguistic context inside the fixed context window, particularly for Devanagari text.

Final Training Recipe

The final model was trained locally on CPU for exactly 2,000 optimizer steps with batch size 128. The optimization system used AdamW with betas (0.9, 0.95), peak learning rate 6e-4, 100 linear warm-up steps, cosine learning-rate decay to 6e-5, weight decay 0.1 on matrix parameters and global gradient clipping at 1.0.

Engineering Conclusions

The largest improvement came from increasing useful token exposure within the fixed optimizer-step budget. Weight tying reduced parameter count but worsened generalization and was therefore rejected. Batch-size improvements showed diminishing returns, motivating a shift toward tokenization efficiency rather than unlimited batch scaling. The hybrid tokenizer produced a further measurable improvement, and the final scheduled AdamW recipe reduced BPB again without changing the model architecture.
